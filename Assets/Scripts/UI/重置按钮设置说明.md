# 重置按钮设置说明

## 功能概述

在游戏右上角创建一个重置按钮，用于重新开始整个场景。

### 主要功能
- 点击显示确认对话框
- 确认后重新加载场景
- 所有数据恢复到初始状态

## 快速设置

### 自动创建（推荐）

1. 在 Unity 菜单栏选择：`橘子树 > UI设置 > 创建右上角重置按钮`
2. 点击"创建重置按钮"
3. 完成！

## 按钮外观

**位置：** 右上角
**样式：** 灰色圆圈 + 白色刷新图标 ↻
**大小：** 50x50 像素

## 工作流程

```
点击重置按钮
  ↓
显示确认对话框
  ↓
用户选择
  ├─ 点击"确定" → 重新加载场景 → 所有数据重置
  └─ 点击"取消" → 关闭对话框 → 继续游戏
```

## 确认对话框

### 外观
- **位置：** 屏幕正中间
- **大小：** 400x200
- **背景：** 深灰色半透明
- **标题：** "确认重置"（黄色）
- **提示：** "确定要重置场景吗？所有进度将会丢失！"

### 按钮
- **确定按钮：** 红色，点击后重置场景
- **取消按钮：** 灰色，点击后关闭对话框

## UI 层级结构

```
Canvas
├── ResetButton（右上角重置按钮）
│   └── Icon (↻)
└── ResetConfirmPanel（确认对话框，初始隐藏）
    ├── Title（标题）
    ├── Message（提示文本）
    └── ButtonContainer（按钮容器）
        ├── YesButton（确定）
        └── NoButton（取消）
```

## 脚本说明

### ResetButtonController.cs

**主要功能：**
- 管理重置按钮的点击事件
- 显示/隐藏确认对话框
- 执行场景重置

**重置方式：**

1. **方法1：重新加载场景（默认）**
   - 使用 `SceneManager.LoadScene()`
   - 完全重置所有对象和数据
   - 推荐使用

2. **方法2：手动重置系统（备用）**
   - 调用各个系统的重置方法
   - 不重新加载场景
   - 适合需要保留某些数据的情况

**公共方法：**
- `ResetSceneDirectly()` - 直接重置场景（跳过确认对话框）

**Inspector 字段：**
- Reset Button - 重置按钮引用
- Confirm Panel - 确认面板引用
- Confirm Yes Button - 确定按钮引用
- Confirm No Button - 取消按钮引用
- Show Confirm Dialog - 是否显示确认对话框

## 重置内容

当点击确定后，以下内容会被重置：

### 使用方法1（重新加载场景）
- ✅ 橘子树生长进度
- ✅ 环境参数（温度、湿度、光照）
- ✅ UI 状态
- ✅ 所有游戏对象状态
- ✅ 时间和计时器

### 使用方法2（手动重置）
需要在 `ResetAllSystems()` 方法中添加：
- 橘子树控制器重置
- 环境管理器重置
- 生长开始控制器重置
- 关闭所有面板

## 自定义修改

### 修改按钮位置

在 `SetupResetButton.cs` 中修改：

```csharp
rect.anchoredPosition = new Vector2(-20, -20); // 右上角偏移
```

### 修改按钮图标

```csharp
text.text = "⟲"; // 使用其他刷新符号
// 可选符号：↻ ⟲ ⟳ ↺ ↶ ⌂
```

### 禁用确认对话框

在 Inspector 中取消勾选 `Show Confirm Dialog`，或在代码中：

```csharp
[SerializeField] private bool showConfirmDialog = false;
```

### 使用手动重置方式

在 `ResetScene()` 方法中注释掉场景加载，启用手动重置：

```csharp
private void ResetScene()
{
    Debug.Log("重置场景...");
    
    // 方法1：重新加载当前场景（推荐）
    // Scene currentScene = SceneManager.GetActiveScene();
    // SceneManager.LoadScene(currentScene.name);
    
    // 方法2：手动重置所有系统
    ResetAllSystems();
}
```

## 常见问题

**Q: 点击重置按钮没反应？**
A: 检查 Inspector 中的按钮引用是否正确设置

**Q: 确认对话框不显示？**
A: 检查 Confirm Panel 引用和 Show Confirm Dialog 选项

**Q: 重置后某些数据没有恢复？**
A: 如果使用手动重置方式，需要在 ResetAllSystems() 中添加对应系统的重置代码

**Q: 想要添加重置动画？**
A: 可以在 ResetScene() 方法中添加淡出效果，然后再加载场景

**Q: 重置按钮位置不对？**
A: 调整 RectTransform 的 Anchored Position

## 扩展建议

1. **添加重置动画**
   ```csharp
   // 淡出效果
   CanvasGroup canvasGroup = GetComponent<CanvasGroup>();
   // 使用 DOTween 或协程实现淡出
   ```

2. **添加音效**
   ```csharp
   AudioSource.PlayClipAtPoint(resetSound, Camera.main.transform.position);
   ```

3. **保存最佳记录**
   ```csharp
   // 在重置前保存最佳成绩
   PlayerPrefs.SetFloat("BestGrowth", currentGrowth);
   ```

4. **添加快捷键**
   ```csharp
   void Update()
   {
       if (Input.GetKeyDown(KeyCode.R))
       {
           OnResetClicked();
       }
   }
   ```

5. **添加重置统计**
   ```csharp
   int resetCount = PlayerPrefs.GetInt("ResetCount", 0);
   PlayerPrefs.SetInt("ResetCount", resetCount + 1);
   ```

## 注意事项

1. 重置会丢失所有未保存的进度
2. 确认对话框可以防止误操作
3. 场景重新加载会有短暂的加载时间
4. 如果场景很大，考虑添加加载提示
5. 测试时确保场景名称正确

## 与其他系统的关系

- **独立系统**：不依赖其他 UI 组件
- **影响范围**：整个场景的所有对象
- **触发时机**：用户主动点击并确认
- **恢复状态**：场景初始状态
