1. **Unity MCP 连接**
   - 如果 Unity MCP 断开连接，Unity 会自动重新连接
   - Node.js 路径已通过 `FixUnityMcpNodePath.cs` 自动配置

2. **场景保存**
   - 清理旧对象后，记得保存场景（Ctrl+S）

3. **配置文件**
   - 新系统的配置文件建议保存在 `Assets/Configs/` 目录

4. **模型引用**
   - 确保所有8个阶段的模型都已正确导入到Unity
   - 模型应该在 `Assets/Prefabs/OrangeTree/` 目录下

## 🔧 故障排除

### 如果编译出错
1. 关闭并重新打开 Unity
2. 菜单：`Assets > Reimport All`
3. 检查 Console 中的具体错误信息

### 如果菜单项不显示
1. 检查脚本是否在 `Assets/Scripts/Editor/` 或 `Assets/Scripts/OrangeTree/Editor/` 目录下
2. 确认脚本没有编译错误
3. 重启 Unity

### 如果模型不显示
1. 检查模型是否已导入
2. 检查模型的 Import Settings
3. 确认模型在 Stages 列表中正确配置
