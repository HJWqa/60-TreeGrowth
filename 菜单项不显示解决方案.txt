🔧 Unity菜单项不显示 - 解决方案
=====================================

如果你在Unity编辑器中看不到"TreePlanQAQ"菜单或"Quick Generate All Orange Trees"选项，
请按照以下步骤操作：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案1：等待Unity编译完成（最常见）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 查看Unity编辑器右下角
2. 如果看到旋转的圆圈图标，说明正在编译
3. 等待编译完成（通常需要10-30秒）
4. 编译完成后，菜单项会自动出现

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案2：强制重新编译
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 在Unity菜单栏选择：Assets → Refresh
2. 或者按快捷键：Ctrl+R (Windows) / Cmd+R (Mac)
3. 等待编译完成

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案3：检查Console错误
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 打开Console窗口（Window → General → Console）
2. 查看是否有编译错误（红色感叹号）
3. 如果有错误，请记录错误信息
4. 常见错误：
   - 命名空间冲突
   - 缺少引用
   - 语法错误

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案4：验证文件位置
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

确认以下文件在正确的位置：

✅ Assets/Scripts/Plant/Editor/OrangeTreeBatchGenerator.cs
✅ Assets/Scripts/Plant/Editor/OrangeTreeGeneratorEditor.cs
✅ Assets/Scripts/Plant/Editor/PlantModelGenerator.cs
✅ Assets/Scripts/Plant/Editor/MenuTest.cs

注意：这些文件必须在"Editor"文件夹中！

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案5：测试菜单项
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 等待编译完成
2. 在Unity顶部菜单栏查找"TreePlanQAQ"
3. 点击后应该看到以下选项：
   - Test Menu Item（测试项）
   - Generate Plant Models
   - Generate Orange Tree Models
   - Quick Generate All Orange Trees
   - Open Orange Trees Folder

4. 先点击"Test Menu Item"测试
5. 如果弹出"测试成功"对话框，说明菜单系统正常

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案6：重启Unity编辑器
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 保存当前场景
2. 关闭Unity编辑器
3. 重新打开项目
4. 等待编译完成
5. 检查菜单项

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案7：检查Unity版本
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

确认Unity版本：
- 推荐：Unity 2022.3 LTS 或更新版本
- 最低：Unity 2020.3 LTS

如果版本太旧，可能不支持某些API

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案8：手动触发编译
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 打开任意一个Editor脚本
2. 添加一个空格然后删除（做一个小修改）
3. 保存文件（Ctrl+S）
4. Unity会自动重新编译

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
方案9：清理Library文件夹（终极方案）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ 注意：这会重新导入所有资源，可能需要较长时间

1. 关闭Unity编辑器
2. 删除项目根目录下的"Library"文件夹
3. 重新打开Unity项目
4. Unity会重新生成Library文件夹
5. 等待导入完成（可能需要几分钟）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
临时替代方案：使用窗口方式
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

如果菜单项始终不显示，可以使用窗口方式：

1. 菜单栏：TreePlanQAQ → Generate Orange Tree Models
2. 在弹出的窗口中点击"生成所有橘子树模型"
3. 这个功能和"Quick Generate"完全相同

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
仍然无法解决？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

请提供以下信息：

1. Unity版本号
2. Console窗口的错误信息（如果有）
3. 是否看到"TreePlanQAQ"菜单
4. 是否看到其他子菜单项
5. "Test Menu Item"是否能正常工作

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
快速检查清单
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

□ Unity编译已完成（右下角没有旋转图标）
□ Console没有红色错误
□ Editor文件夹中有所有必需的脚本
□ 已尝试Assets → Refresh
□ 已尝试重启Unity

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

如果以上所有方案都尝试过了，菜单项应该会出现。
最常见的原因是Unity还在编译中，请耐心等待！⏳
