# 配置手工模型到生长系统

## 当前状态
✅ 已移除代码中的Dormant和Dead阶段
✅ 已删除程序化生成模型的代码
✅ 手工模型已导入Unity并放置在场景中

## 需要在Unity编辑器中完成的配置

### 1. 清理场景中的旧对象
在Hierarchy中删除以下对象（如果存在）：
- `OrangeTreeGenerator`
- `OrangeTreePreview_Test`

### 2. 配置PlantGrowthController

找到场景中带有PlantGrowthController组件的GameObject，然后：

#### 步骤1: 移除Dormant和Dead配置
在Inspector中找到`Stage Configs`列表，删除最后两项：
- Stage 8: Dormant
- Stage 9: Dead

现在应该只剩下8个阶段。

#### 步骤2: 确认8个阶段的模型引用
确保每个阶段都正确引用了你的手工模型：

| Stage | Stage Name | Stage Model (拖拽GameObject) |
|-------|-----------|------------------------------|
| 0 | Seed | OrangeTree_Seed |
| 1 | Sprout | OrangeTree_Sprout |
| 2 | Seedling | OrangeTree_Seeding |
| 3 | YoungTree | OrangeTree_YoungTreetree |
| 4 | MatureTree | OrangeTree_Maturetree |
| 5 | Flowering | OrangeTree_Flowering |
| 6 | Fruiting | OrangeTree_Fruting |
| 7 | Harvest | OrangeTree_Harvest |

#### 步骤3: 配置Growth Config
如果还没有配置，需要：
1. 在Inspector中找到`Growth Config`字段
2. 拖拽`Assets/Configs/OrangeTreeGrowthConfig.asset`到这个字段

### 3. 初始化设置
确保PlantGrowthController的初始状态：
- `Current Growth`: 0
- `Current Stage`: Seed (0)

### 4. 测试生长系统

#### 方法1: 使用EnvironmentSimulator
1. 找到场景中的EnvironmentSimulator对象
2. 调整环境参数（温度、湿度、阳光）
3. 点击Play，观察树的生长

#### 方法2: 手动测试
1. 选中带有PlantGrowthController的GameObject
2. 在Inspector中找到`Current Growth`
3. 在Play模式下，手动修改这个值（0-100）
4. 观察树的阶段变化：
   - 0-10: Seed
   - 10-25: Sprout
   - 25-50: Seedling
   - 50-75: YoungTree
   - 75-90: MatureTree
   - 90-95: Flowering
   - 95-100: Fruiting/Harvest

### 5. 调整过渡动画
如果觉得阶段切换太快或太慢，可以调整每个阶段的`Transition Duration`：
- 默认值: 1秒
- 建议值: 1-3秒（根据视觉效果调整）

## 常见问题

### Q: 模型切换时没有动画效果？
A: 检查`Transition Duration`是否大于0，建议设置为1-2秒。

### Q: 树不生长？
A: 检查：
1. Growth Config是否已配置
2. EnvironmentSimulator是否在运行
3. 环境参数是否在合理范围内（温度15-35°C，湿度30-90%）

### Q: 某个阶段的模型没有显示？
A: 检查：
1. 该阶段的Stage Model是否正确引用了GameObject
2. GameObject是否在场景中（不要删除）
3. GameObject的初始状态应该是Inactive（系统会自动激活）
