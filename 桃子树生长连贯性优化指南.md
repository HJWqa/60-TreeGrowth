# 桃子树生长连贯性优化指南

## 🎯 什么是生长连贯性？

**连贯性** = 相邻阶段之间的视觉过渡自然流畅，没有突兀的跳跃

**问题示例**：
- ❌ 阶段3是小苗，阶段4突然变成大树（跳跃太大）
- ❌ 叶子数量从10片突然变成100片（不自然）
- ❌ 树干粗细变化不一致
- ❌ 颜色突变

**目标**：
- ✅ 每个阶段看起来像是前一阶段"长大了一点"
- ✅ 尺寸、数量、颜色渐进变化
- ✅ 结构保持一致性

## 📐 方法一：使用参考比例系统（推荐）

### 核心原则：建立统一的比例关系

创建一个**比例参考表**，确保每个阶段按固定比例增长：

| 阶段 | 名称 | 高度比例 | 宽度比例 | 叶子数量 | 果实数量 |
|------|------|---------|---------|---------|---------|
| 1 | Seed | 0.1 | 0.1 | 0 | 0 |
| 2 | Sprout | 0.2 | 0.15 | 2 | 0 |
| 3 | Seedling | 0.35 | 0.25 | 6 | 0 |
| 4 | YoungTree | 0.55 | 0.45 | 20 | 0 |
| 5 | MatureTree | 0.8 | 0.7 | 50 | 0 |
| 6 | Flowering | 0.9 | 0.8 | 60 | 0 (25朵花) |
| 7 | Fruiting | 1.0 | 0.9 | 70 | 20 |
| 8 | Harvest | 1.0 | 1.0 | 80 | 30 |

### 在Blender中应用比例系统

#### 步骤1：设置参考对象

1. 在场景中创建一个参考立方体：
   - `Shift + A` → Mesh → Cube
   - 命名为 `Reference_Scale`
   - 设置为最终成熟树的高度（比如2米）
   - 锁定它：右键 → Lock → All Transforms

2. 创建高度标记：
   - 在不同高度添加空对象作为标记
   - 0.2米、0.35米、0.55米、0.8米等
   - 命名为 `Height_Stage2`、`Height_Stage3` 等

#### 步骤2：逆向建模时对照参考

在创建每个阶段时：

1. **打开参考文件**
   - 保持阶段8（成熟期）在一个窗口
   - 新阶段在另一个窗口
   - 或使用 Window → New Window 分屏对比

2. **使用测量工具**
   - 按 `N` 打开侧边栏
   - View → Measure
   - 测量关键尺寸确保比例正确

3. **对齐到参考高度**
   - 选择树的顶部
   - 按 `G` → `Z` 移动
   - 对齐到对应的高度标记

## 🎨 方法二：渐进式修改法（最佳实践）

### 核心思路：每次只改变一点点

#### 阶段8 → 阶段7（结果期）

**改变内容**：
- ✅ 桃子颜色：粉红 → 青绿
- ✅ 桃子数量：30个 → 20个（删除33%）
- ✅ 桃子大小：100% → 70%
- ❌ 不改变：树干、树枝、叶子

**Blender操作**：
```
1. 另存为新文件
2. 只修改桃子：
   - 选择所有桃子
   - 材质颜色改为青绿色
   - 随机删除10个桃子（保持分布均匀）
   - 缩放到0.7
3. 其他部分完全不动
4. 保存
```

#### 阶段7 → 阶段6（开花期）

**改变内容**：
- ✅ 删除所有桃子
- ✅ 添加25朵花
- ✅ 叶子数量：70 → 60（减少14%）
- ❌ 不改变：树干、树枝结构

**Blender操作**：
```
1. 删除桃子
2. 在相同位置添加花朵
3. 轻微减少叶子（如果是粒子系统，改数量）
4. 保存
```

#### 阶段6 → 阶段5（成树期）

**改变内容**：
- ✅ 删除所有花朵
- ✅ 叶子数量：60 → 50（减少17%）
- ❌ 不改变：树干、树枝

#### 阶段5 → 阶段4（小树期）

**改变内容**：
- ✅ 删除部分树枝（保留主要的3-4根）
- ✅ 整体缩放：100% → 70%
- ✅ 叶子数量：50 → 20（减少60%）
- ⚠️ 关键：保持树枝的相对位置和角度

**Blender操作**：
```
1. 进入编辑模式
2. 识别主要树枝（最粗的几根）
3. 只删除细小的分支
4. 退出编辑模式
5. 整体缩放到0.7
6. 减少叶子数量
```

#### 阶段4 → 阶段3（幼苗期）

**改变内容**：
- ✅ 删除所有树枝
- ✅ 保留主干
- ✅ 主干缩短：70% → 35%
- ✅ 主干变细：70% → 30%
- ✅ 叶子：20片 → 6片（只在顶部）

**关键技巧**：
```
保持主干的底部位置不变
只缩短顶部
这样看起来像是"还没长高"而不是"缩小了"
```

#### 阶段3 → 阶段2（发芽期）

**改变内容**：
- ✅ 主干变成细茎
- ✅ 高度：35% → 20%
- ✅ 叶子：6片 → 2片
- ✅ 颜色：深绿 → 浅绿（嫩芽）

#### 阶段2 → 阶段1（种子期）

**改变内容**：
- ✅ 完全不同的形态（种子）
- ⚠️ 这是唯一允许的"跳跃"

## 🔄 方法三：在Unity中添加过渡动画

即使模型阶段分明，也可以在Unity中添加平滑过渡：

### 1. 交叉淡入淡出（Cross-Fade）

创建脚本：`Assets/Scripts/Plant/SmoothStageTransition.cs`

```csharp
using UnityEngine;
using System.Collections;

namespace TreePlanQAQ.Plant
{
    public class SmoothStageTransition : MonoBehaviour
    {
        public GameObject[] stageModels; // 所有阶段模型
        public float transitionDuration = 2f; // 过渡时间
        
        private int currentStage = 0;
        private int targetStage = 0;
        
        void Start()
        {
            // 初始化：只显示第一个阶段
            for (int i = 0; i < stageModels.Length; i++)
            {
                stageModels[i].SetActive(i == 0);
            }
        }
        
        public void TransitionToStage(int newStage)
        {
            if (newStage == currentStage || newStage >= stageModels.Length)
                return;
                
            targetStage = newStage;
            StartCoroutine(SmoothTransition());
        }
        
        private IEnumerator SmoothTransition()
        {
            GameObject oldModel = stageModels[currentStage];
            GameObject newModel = stageModels[targetStage];
            
            // 激活新模型
            newModel.SetActive(true);
            
            // 获取所有Renderer
            Renderer[] oldRenderers = oldModel.GetComponentsInChildren<Renderer>();
            Renderer[] newRenderers = newModel.GetComponentsInChildren<Renderer>();
            
            // 新模型从透明开始
            SetAlpha(newRenderers, 0f);
            
            float elapsed = 0f;
            
            while (elapsed < transitionDuration)
            {
                elapsed += Time.deltaTime;
                float t = elapsed / transitionDuration;
                
                // 旧模型淡出
                SetAlpha(oldRenderers, 1f - t);
                
                // 新模型淡入
                SetAlpha(newRenderers, t);
                
                yield return null;
            }
            
            // 完成过渡
            SetAlpha(oldRenderers, 0f);
            SetAlpha(newRenderers, 1f);
            oldModel.SetActive(false);
            
            currentStage = targetStage;
        }
        
        private void SetAlpha(Renderer[] renderers, float alpha)
        {
            foreach (Renderer renderer in renderers)
            {
                foreach (Material mat in renderer.materials)
                {
                    Color color = mat.color;
                    color.a = alpha;
                    mat.color = color;
                    
                    // 设置渲染模式为透明
                    mat.SetFloat("_Mode", 3);
                    mat.SetInt("_SrcBlend", (int)UnityEngine.Rendering.BlendMode.SrcAlpha);
                    mat.SetInt("_DstBlend", (int)UnityEngine.Rendering.BlendMode.OneMinusSrcAlpha);
                    mat.SetInt("_ZWrite", 0);
                    mat.DisableKeyword("_ALPHATEST_ON");
                    mat.EnableKeyword("_ALPHABLEND_ON");
                    mat.DisableKeyword("_ALPHAPREMULTIPLY_ON");
                    mat.renderQueue = 3000;
                }
            }
        }
    }
}
```

### 2. 缩放过渡

```csharp
using UnityEngine;
using System.Collections;

namespace TreePlanQAQ.Plant
{
    public class ScaleTransition : MonoBehaviour
    {
        public GameObject[] stageModels;
        public float transitionDuration = 1.5f;
        
        private int currentStage = 0;
        
        public void TransitionToStage(int newStage)
        {
            if (newStage == currentStage) return;
            StartCoroutine(ScaleTransitionCoroutine(newStage));
        }
        
        private IEnumerator ScaleTransitionCoroutine(int newStage)
        {
            GameObject oldModel = stageModels[currentStage];
            GameObject newModel = stageModels[newStage];
            
            // 新模型从小开始
            newModel.SetActive(true);
            newModel.transform.localScale = Vector3.zero;
            
            float elapsed = 0f;
            
            while (elapsed < transitionDuration)
            {
                elapsed += Time.deltaTime;
                float t = elapsed / transitionDuration;
                
                // 使用缓动函数
                float easeT = EaseOutCubic(t);
                
                // 旧模型缩小
                oldModel.transform.localScale = Vector3.Lerp(Vector3.one, Vector3.zero, easeT);
                
                // 新模型放大
                newModel.transform.localScale = Vector3.Lerp(Vector3.zero, Vector3.one, easeT);
                
                yield return null;
            }
            
            oldModel.SetActive(false);
            oldModel.transform.localScale = Vector3.one;
            newModel.transform.localScale = Vector3.one;
            
            currentStage = newStage;
        }
        
        private float EaseOutCubic(float t)
        {
            return 1f - Mathf.Pow(1f - t, 3f);
        }
    }
}
```

### 3. 混合过渡（最佳效果）

结合淡入淡出和缩放：

```csharp
using UnityEngine;
using System.Collections;

namespace TreePlanQAQ.Plant
{
    public class BlendedTransition : MonoBehaviour
    {
        public GameObject[] stageModels;
        public float transitionDuration = 2f;
        public AnimationCurve scaleCurve = AnimationCurve.EaseInOut(0, 0, 1, 1);
        public AnimationCurve alphaCurve = AnimationCurve.Linear(0, 0, 1, 1);
        
        private int currentStage = 0;
        
        public void TransitionToStage(int newStage)
        {
            if (newStage == currentStage) return;
            StartCoroutine(BlendedTransitionCoroutine(newStage));
        }
        
        private IEnumerator BlendedTransitionCoroutine(int newStage)
        {
            GameObject oldModel = stageModels[currentStage];
            GameObject newModel = stageModels[newStage];
            
            newModel.SetActive(true);
            
            // 新模型初始状态
            Vector3 oldScale = oldModel.transform.localScale;
            Vector3 newTargetScale = newModel.transform.localScale;
            newModel.transform.localScale = oldScale * 0.8f; // 从稍小开始
            
            Renderer[] oldRenderers = oldModel.GetComponentsInChildren<Renderer>();
            Renderer[] newRenderers = newModel.GetComponentsInChildren<Renderer>();
            
            SetAlpha(newRenderers, 0f);
            
            float elapsed = 0f;
            
            while (elapsed < transitionDuration)
            {
                elapsed += Time.deltaTime;
                float t = elapsed / transitionDuration;
                
                float scaleT = scaleCurve.Evaluate(t);
                float alphaT = alphaCurve.Evaluate(t);
                
                // 缩放过渡
                oldModel.transform.localScale = Vector3.Lerp(oldScale, oldScale * 0.8f, scaleT);
                newModel.transform.localScale = Vector3.Lerp(oldScale * 0.8f, newTargetScale, scaleT);
                
                // 透明度过渡
                SetAlpha(oldRenderers, 1f - alphaT);
                SetAlpha(newRenderers, alphaT);
                
                yield return null;
            }
            
            // 恢复状态
            oldModel.SetActive(false);
            oldModel.transform.localScale = oldScale;
            newModel.transform.localScale = newTargetScale;
            SetAlpha(newRenderers, 1f);
            
            currentStage = newStage;
        }
        
        private void SetAlpha(Renderer[] renderers, float alpha)
        {
            foreach (Renderer renderer in renderers)
            {
                foreach (Material mat in renderer.materials)
                {
                    Color color = mat.color;
                    color.a = alpha;
                    mat.color = color;
                    
                    // 设置透明渲染模式
                    if (alpha < 1f)
                    {
                        mat.SetFloat("_Mode", 3);
                        mat.SetInt("_SrcBlend", (int)UnityEngine.Rendering.BlendMode.SrcAlpha);
                        mat.SetInt("_DstBlend", (int)UnityEngine.Rendering.BlendMode.OneMinusSrcAlpha);
                        mat.SetInt("_ZWrite", 0);
                        mat.EnableKeyword("_ALPHABLEND_ON");
                        mat.renderQueue = 3000;
                    }
                }
            }
        }
    }
}
```

## 🎬 方法四：使用Shape Keys（形态键）

### 在Blender中创建形态键动画

这是最高级的方法，可以实现真正的连续变形：

#### 步骤1：准备基础模型

1. 打开阶段1（种子）模型
2. 确保所有对象合并为一个Mesh
3. Object → Join（或 `Ctrl + J`）

#### 步骤2：添加形态键

1. 选择模型
2. 右侧属性面板 → Object Data Properties（绿色三角形图标）
3. Shape Keys 部分
4. 点击 `+` 添加 Basis（基础形态）
5. 再点击 `+` 添加 Key 1，命名为 "Stage2_Sprout"

#### 步骤3：编辑形态键

1. 确保 "Stage2_Sprout" 被选中
2. Value 设为 1.0
3. 进入编辑模式 `Tab`
4. 修改模型使其变成阶段2的样子：
   - 拉长茎
   - 添加叶子（挤出）
   - 调整形状
5. 退出编辑模式

#### 步骤4：添加更多形态键

重复步骤2-3，为每个阶段创建形态键：
- Stage3_Seedling
- Stage4_YoungTree
- Stage5_MatureTree
- 等等

#### 步骤5：导出动画

1. 切换到 Animation 工作区
2. 在时间轴上设置关键帧：
   - 帧1：所有形态键 = 0（种子）
   - 帧100：Stage2 = 1（发芽）
   - 帧200：Stage3 = 1（幼苗）
   - 等等
3. File → Export → FBX
4. 勾选 ✓ Baked Animation
5. 导出到Unity

**注意**：这个方法非常复杂，只适合高级用户。

## 📊 连贯性检查清单

在完成所有阶段后，使用这个清单检查：

### 视觉连贯性
- [ ] 相邻阶段的高度差不超过50%
- [ ] 相邻阶段的宽度差不超过50%
- [ ] 树干粗细渐进变化
- [ ] 叶子数量渐进变化（不要突然从10变成100）
- [ ] 颜色变化平滑（没有突变）

### 结构连贯性
- [ ] 主干的位置和角度保持一致
- [ ] 主要树枝的生长方向一致
- [ ] 树冠的整体形状相似
- [ ] 根部位置固定

### 数量连贯性
- [ ] 叶子数量：2 → 6 → 20 → 50 → 60 → 70 → 80
- [ ] 树枝数量：0 → 0 → 0 → 4 → 6 → 6 → 6
- [ ] 花朵/果实：0 → 0 → 0 → 0 → 0 → 25 → 20 → 30

### 比例连贯性
- [ ] 每个阶段相对于前一阶段的增长比例合理
- [ ] 使用参考对象验证尺寸
- [ ] 在Unity中并排放置检查

## 💡 专业技巧

### 技巧1：使用"中间帧"

如果两个阶段之间跳跃太大，添加一个中间阶段：
- 阶段4和5之间 → 添加阶段4.5
- 这样从8个阶段变成10个或12个阶段

### 技巧2：使用粒子系统的动画

对于叶子和果实，使用粒子系统的Birth和Death：
```
在Unity中：
- 叶子粒子系统：Start Lifetime = 生长阶段时长
- 逐渐增加Emission Rate
- 这样叶子会"逐渐长出来"而不是"突然出现"
```

### 技巧3：使用Shader实现生长效果

创建一个生长Shader，通过调整参数实现平滑过渡：
```glsl
// 伪代码
float growthProgress = 0.0 to 1.0;
vertex.position.y *= growthProgress;
color.a = growthProgress;
```
